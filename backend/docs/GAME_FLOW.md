# Игровой процесс

## Полный игровой цикл

```
1. Матчмейкинг
   ↓
2. Countdown (3-2-1)
   ↓
3. Фаза планирования (20 сек)
   ↓
4. Фаза боя (автоматическая)
   ↓
5. Результаты раунда
   ↓
6. Следующий раунд
   ↓
7. Победа (когда HP одного игрока <= 0)
```

---

## 1. Матчмейкинг

### Как работает:

1. Игрок отправляет `POST /matchmaking/join`
2. Сервер добавляет игрока в очередь
3. Когда находится 2 игрока → создается игра
4. Оба игрока получают `game_id`
5. Игроки подключаются через WebSocket

### Начальное состояние:

- **HP:** 100 (каждый игрок)
- **Монеты:** 10 (каждый игрок)
- **Раунд:** 0 (перед началом)

---

## 2. Countdown

После подключения обоих игроков начинается обратный отсчет:

```json
{"type": "countdown", "seconds": 3}
{"type": "countdown", "seconds": 2}
{"type": "countdown", "seconds": 1}
```

**Длительность:** 3 секунды

**Цель:** Дать игрокам время подготовиться

---

## 3. Фаза планирования

### Начало фазы:

```json
{
  "type": "planning_phase_start",
  "round": 1,
  "time_left": 20
}
```

### Что можно делать:

1. **Покупать юнитов** - тратить монеты на новых юнитов
2. **Перемещать юнитов** - между полем и скамейкой
3. **Продавать юнитов** - получить 50% возврат
4. **Мерджить юнитов** - объединять одинаковых

### Ограничения:

- **Время:** 20 секунд
- **Максимум юнитов на поле:** 6
- **Максимум юнитов на скамейке:** 4
- **Всего юнитов:** 10

### Монеты:

- **Начальные:** 10 монет
- **За каждый раунд:** +5 монет
- **Стоимость юнитов:** берется из БД (warrior ≈ 3, archer ≈ 4, mage ≈ 5)

### Автомердж:

Если у игрока 2 одинаковых юнита (тип + уровень) → автоматически сливаются в 1 юнита уровня +1.

**Пример:**
```
2x warrior lvl 1 → 1x warrior lvl 2
2x warrior lvl 2 → 1x warrior lvl 3
2x warrior lvl 3 → 1x warrior lvl 4 (максимум)
```

**Характеристики при мердже:**
```
HP = base_hp * 2^(level-1)
Attack = base_attack * 2^(level-1)

Пример warrior:
lvl 1: HP=100, Attack=20
lvl 2: HP=200, Attack=40
lvl 3: HP=400, Attack=80
lvl 4: HP=800, Attack=160
```

---

## 4. Фаза боя

### Начало боя:

```json
{
  "type": "battle_phase_start",
  "round": 1
}
```

### Симуляция:

Бой симулируется на сервере:
- Юниты автоматически ищут врагов
- Двигаются к ним
- Атакуют когда в радиусе
- Бой идет до тех пор пока все юниты одной стороны не умрут

### События боя:

Сервер отправляет все события боя для воспроизведения:

```json
{
  "type": "battle_events",
  "events": [
    {"time": 0.0, "type": "battle_start"},
    {"time": 0.1, "type": "movement", "unit_id": "...", "position": [2.7, 1.8]},
    {"time": 1.5, "type": "attack", "unit_id": "...", "target_id": "...", "damage": 20},
    {"time": 3.5, "type": "death", "unit_id": "..."},
    {"time": 3.5, "type": "battle_end"}
  ]
}
```

**Клиент должен:**
1. Получить события
2. Воспроизвести их по времени (event.time * 1000 мс)
3. Показать анимации движения, атак, смертей

### Длительность:

- **Симуляция:** (зависит от длительности сражения)
- **Воспроизведение:** длительность + 2 секунды
- **Пауза:** Сервер ждет пока клиент воспроизведет бой

---

## 5. Результаты раунда

### Конец боя:

```json
{
  "type": "battle_phase_end",
  "round": 1,
  "round_winner": "player1",
  "damage_to_player1": 0,
  "damage_to_player2": 3,
  "player1_hp": 100,
  "player2_hp": 97
}
```

### Расчет урона:

**Если один игрок победил:**
```
Урон = количество_выживших_юнитов + 1
```

**Пример:**
- Player1: 2 юнита выжили
- Player2: 0 юнитов выжили
- Урон Player2: 2 + 1 = 3 HP

**Если ничья (все юниты обеих сторон мертвы):**
```
Урон обоим = 1 HP
```

---

## 6. Следующий раунд

После результатов раунда:

1. **Если игра не закончена** → начинается новая фаза планирования
2. **Монеты:** +5 монет каждому игроку
3. **Раунд:** +1

```json
{
  "type": "planning_phase_start",
  "round": 2,
  "time_left": 20
}
```

---

## 7. Победа

### Условие победы:

Игра заканчивается когда HP одного или обоих игроков <= 0.

### Game Over:

```json
{
  "type": "game_over",
  "winner": "player1",
  "player1_hp": 15,
  "player2_hp": -3
}
```

**Возможные значения winner:**
- `"player1"` - Player 1 победил
- `"player2"` - Player 2 победил
- `"draw"` - Ничья (оба HP <= 0 одновременно)

### После победы:

1. Игра удаляется из Redis (через 5 секунд)
2. Игроки освобождаются (могут начать новую игру)
3. WebSocket соединение закрывается

---

## Временная шкала (пример)

```
t=0s    : Матчмейкинг
t=1s    : Игра создана, countdown начался
t=4s    : Фаза планирования раунд 1 (20 сек)
t=24s   : Фаза боя раунд 1 (5 сек симуляция + 7 сек воспроизведение)
t=36s   : Результаты раунд 1
t=36s   : Фаза планирования раунд 2 (20 сек)
t=56s   : Фаза боя раунд 2
...
t=300s  : Game over (после ~10-15 раундов)
```
